Worklog Tuần 6

Mục tiêu tuần 6:

  • Làm chủ nguyên tắc cấp quyền tối thiểu (Least Privilege) thông qua việc cấu hình điều kiện ràng buộc (Condition), đặt ranh giới quyền lực tối đa (Permission Boundary) và thiết lập hệ thống đăng nhập một lần (Single Sign-On) trên môi trường đa tài khoản doanh nghiệp.
  • Hiểu cách ứng dụng thẻ nhãn (Tag) để phân loại tài nguyên khoa học và viết các chính sách bảo mật dựa trên Tag để tự động hóa phân quyền trên quy mô lớn.
  • Bật và vận hành trung tâm quản lý trạng thái bảo mật để tự động rà quét, phát hiện các lỗ hổng cấu hình sai sót theo tiêu chuẩn quốc tế.
  • Ứng dụng kiến trúc Serverless để lập lịch chu kỳ bật/tắt máy chủ tự động và tích hợp ChatOps gửi thông báo thời gian thực về kênh làm việc doanh nghiệp.

Kết quả đạt được tuần 6:

  • Làm chủ kỹ thuật phân quyền siết chặt tầng sâu (IAM Advanced):

    • Cấu hình thành công block Condition dựa trên địa chỉ IP tĩnh (aws:SourceIp) và mốc thời gian hệ thống (aws:CurrentTime), giúp tước quyền truy cập của các tài khoản kiểm toán ngay khi hết hạn dự án hoặc truy cập ngoài văn phòng.
    • Vận dụng thành công Permissions Boundary để gán “vòng kim cô” giới hạn quyền tối đa cho user phụ con, triệt tiêu hoàn toàn nguy cơ leo thang đặc quyền trái phép dù user đó được gán policy Admin rộng.
    • Ứng dụng thành công giải pháp EC2 Instance Profile, cấp quyền cho mã nguồn Python truy cập S3 qua IAM Role, loại bỏ hoàn toàn việc hardcode Access Key nguy hiểm vào source code.

  • Triển khai Landing Zone và Đăng nhập một lần (SSO):

    • Kích hoạt thành công AWS Organizations và cấu hình cổng thông tin tập trung AWS IAM Identity Center trên mô hình group-based permissions.
    • Vận hành mượt mà các bộ quyền hạn (Permission Sets) kết nối tài khoản doanh nghiệp; thành thạo việc sử dụng AWS CLI xác thực tự động qua lệnh aws configure sso và thoát phiên an toàn bằng aws sso logout.
    • Tự động hóa hoàn toàn quy trình truy vấn, quản lý và tạo User/Group số lượng lớn thông qua việc lập trình tương tác với Identity Store APIs bằng thư viện Python Boto3.

  • Phân loại tài nguyên bằng Tag và Đánh giá an ninh qua Security Hub:

    • Thực hiện gán nhãn, lọc tài nguyên và tự động hóa gom nhóm máy ảo theo thẻ (Tag-based group) thành công vào Resource Group MarketingBU bằng cả Console và AWS CLI.
    • Vận hành thành công AWS Config phối hợp với AWS Security Hub để chạy rà quét cấu hình tự động theo tiêu chuẩn quốc tế AWS Foundational Security Best Practices và PCI DSS.
    • Làm quen với vai trò Kiểm toán viên, đọc hiểu đồ thị rủi ro (Findings) và thực hành tắt/khắc phục các cấu hình sai sót để tối ưu điểm số an ninh trên Dashboard tổng quan.

  • Hiện thực hóa ChatOps tối ưu chi phí EC2 với Lambda:

    • Xây dựng thành công hệ thống tự động hóa Serverless: Viết mã nguồn Python Boto3 trên AWS Lambda nhận diện chính xác thẻ Tag hệ thống để thực thi chu kỳ hạ ngầm hoặc kích hoạt máy chủ EC2.
    • Tích hợp thành công dịch vụ Amazon EventBridge làm trigger hẹn giờ bật/tắt server tự động, giúp doanh nghiệp cắt giảm hơn 60% chi phí lãng phí của các môi trường Test/Staging.
    • Thiết lập thành công luồng kết nối Webhook gửi thông điệp trạng thái máy chủ đổi màu trực quan (Stopping/Starting) về kênh thông báo Slack theo thời gian thực.
