### Week 7 Objectives:

- Thoroughly understood server-side encryption mechanisms using customer-managed keys (AWS KMS CMK) to protect data on Amazon S3.
- Mastered auditing and analysis solutions for encryption API call history by coordinating the triad of AWS CloudTrail, Amazon S3, and Amazon Athena.
- Gained a deep understanding of how to design secure isolated network environments, configuring Security Groups and Multi-AZ database subnet groups (DB Subnet Groups) for Amazon RDS.
- Practiced deploying a backend application (Node.js) running on EC2 that performs real-time data read/write operations down to the RDS MySQL database tier.
- Explored the roadmap for heterogeneous data migration from large systems like SQL Server and Oracle to Amazon Aurora MySQL using the toolset combination of AWS SCT and AWS DMS.

### Week 7 Achievements:

- Completing the Security Layer with AWS KMS:
  - Successfully initialized a symmetric Customer Managed Key (kms-s3-key) and configured automated key rotation policies to enhance overall system security.
  - Successfully implemented encryption for all files uploaded to S3; mastered secure data sharing within a limited time frame using Presigned URL temporary access links.
  - Successfully built an automated auditing system using CloudTrail and executed direct SQL queries on Amazon Athena to extract successful KMS key invocation logs.

- Deploying Full-Stack Applications Connected to Amazon RDS:
  - Established a complete Multi-AZ network infrastructure: segregated isolated private subnets for the database, and configured strict Inbound rules on the RDS port 3306 to accept traffic exclusively from the EC2 server's Security Group.
  - Successfully deployed Node.js backend source code, configuring connection parameters through a .env environment file pointing directly to the unique RDS MySQL Endpoint.
  - Operated the data management lifecycle: created databases, tables, and seeded sample records; configured automated backups, set up CloudWatch alarms, and successfully practiced snapshot restoration to recover from simulated outages.

- Mastery of Enterprise Data Migration Solutions:
  - Mastered connection and administration workflows for Windows Server and Linux systems via specialized RDP Clients and secure Systems Manager (SSM) Fleet Manager without opening public ports.
  - Built a sharp analytical framework for schema conversion (SCT) and data replication (DMS) processes: understanding schema extraction, foreign key handling, constraints dropping, and endpoint configuration.
  - Explored modern cloud migration architectures using AWS DMS Serverless to automatically scale compute resources based on workload demands, integrated with automated status notifications via SNS.
  - Developed advanced troubleshooting competencies through deep dives into real-world scenarios: resolving replication instance memory pressure and fixing table synchronization and data type mismatch errors.
